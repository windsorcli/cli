# Windsor state directory

`.windsor/` lives at the project root. The runtime creates and updates
everything inside it; operators should not edit these files by hand ‚Äî
content here is regenerated by the next command that touches it.

## Layout

```
.windsor/
‚îú‚îÄ‚îÄ context                                              current context name (single-line text)
‚îú‚îÄ‚îÄ .build-id                                            per-day build identifier
‚îú‚îÄ‚îÄ .session.<token>                                     shell-session reset markers
‚îú‚îÄ‚îÄ cache/oci/<extraction-key>/                          OCI artifact extraction cache
‚îú‚îÄ‚îÄ contexts/<context-name>/
‚îÇ   ‚îú‚îÄ‚îÄ workstation.yaml                                 system-managed workstation state
‚îÇ   ‚îú‚îÄ‚îÄ terraform/<component-id>/terraform.tfvars        generated Terraform variables
‚îÇ   ‚îú‚îÄ‚îÄ .terraform/                                      terraform init working directory
‚îÇ   ‚îî‚îÄ‚îÄ .tfstate/                                        local-backend state cache
‚îî‚îÄ‚îÄ plan/<kustomization-name>/kustomization.yaml         ephemeral kustomization (flux diff)
```

## Project-root files

| Path | Type | Description |
|------|------|-------------|
| `context` | text | The current context name, one line. Read by every command that resolves the active context; written by `windsor set context`. |
| `.build-id` | text | Per-day build identifier emitted as `WINDSOR_BUILD_ID`. Format: `YYMMDD.<random>.<counter>`. Updated by `windsor build-id --new`. |
| `.session.<token>` | empty marker | Shell-session reset signal. Created when the runtime needs the next hook tick to re-export env; consumed by the shell hook. |
| `cache/oci/<key>/` | directory | Extracted contents of OCI artifacts (blueprints, modules). Persisted across runs; re-extraction is cheap on cache hit. |
| `plan/<name>/kustomization.yaml` | YAML | Synthetic kustomization written during `flux diff` for kustomizations that are not yet deployed. Deleted at the end of the diff run. |

## `contexts/<context-name>/`

| Path | Type | Description |
|------|------|-------------|
| `workstation.yaml` | YAML | System-managed workstation state ‚Äî VM address, runtime driver, DNS address/forwarders, provisioned flags. Distinct from operator-authored `contexts/<name>/values.yaml`. Written by the runtime during `init`/`bootstrap`/`configure`. |
| `terraform/<component-id>/terraform.tfvars` | HCL | Generated Terraform variables for one blueprint component. All input expressions are evaluated before write. Overwritten on every `plan`/`apply`/`show terraform`. |
| `.terraform/` | directory | Terraform's own working directory (provider plugins, module cache). Created by `terraform init`; cleaned by `windsor init --reset`. |
| `.tfstate/` | directory | Local-backend Terraform state cache. Present when `terraform.backend.type=local`; cleaned by `windsor init --reset`. |
